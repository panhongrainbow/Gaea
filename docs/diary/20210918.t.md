 # 2021年09月18日 開發日誌

> - 為了開放方便，會模擬數據庫回傳，因為有一個狀況，開發者同時會用 IDE 工具開啟二個不同的專案，現在大家可能用 docker 容器 或 K8S 去模擬開發環境，但是像複雜的專案，模擬環境可能因為筆電硬體性能的限制，無法同時模擬多種開發環境，這時模擬數據庫回傳可以幫助開發者進行臨時簡易的開發環境
> - 又有另一個狀況，可能開發者可能會出差或旅行，可能路上會發一些交通工具，這時會希望打開專案時，就能進行臨時開發，這時模擬數據庫回傳會對開發者很方便，因為 docker 容器 或 K8S 去模擬開發環境也要花費時間去啟動，對於在旅途的開發者不是很適合

## [問題和解決] 在模擬數據庫回傳時，發現錯誤

這時，有寫三個函式在模擬數據庫資料回傳

| 項目 | 函式名稱                 | 說明                                                   |
| ---- | ------------------------ | ------------------------------------------------------ |
| 1    | SelectNovelResult()      | 回傳 29 本小說模擬資料                                 |
| 2    | MakeNovelEmptyResult()   | 模擬回傳空數據庫資料，此時數據庫沒有任何一本小說的數據 |
| 3    | InsertFirstNovelResult() | 模擬插入第一本小說的數據                               |

<img src="../assets/panhongrainbow/image-20210919232249390.png" alt="image-20210919232249390" style="zoom:80%;" /> 

- 一開始使用 SelectNovelResult() 函式去模擬數據庫回傳，沒有錯誤發生，因為是直接由真實的 MariaDB 數據庫抄出來的數據，相對比較不會有問題
- 接下來自行編寫 MakeNovelEmptyResult() 和 InsertFirstNovelResult() 就發現問題，一直會發生錯誤，最後再把 ResultSet 整個完整的數據從 SelectNovelResult() 複制到 MakeNovelEmptyResult() 後，之後就正常

<img src="../assets/panhongrainbow/image-20210919234749545.png" alt="image-20210919234749545" style="zoom:80%;" />
